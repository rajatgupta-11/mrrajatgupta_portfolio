Executive Summary

[cite_start]This 250 mm testbed investigates robust PID stabilization and deterministic failsafe behavior[cite: 4]. All flight-critical control logic executes directly on the Brahma F4 MK-III flight controller [cite_start]to guarantee hard real-time execution[cite: 6, 84], while non-real-time autonomy experiments run on a [cite_start]Raspberry Pi companion computer[cite: 7, 85].

System Architecture

Control Strategy

[cite_start]Independent PID loops stabilize roll, pitch, and yaw[cite: 42]. Tuning prioritized disturbance rejection and vibration tolerance through conservative [cite_start]derivative gains and dynamic notch filtering[cite: 46, 52].

Deterministic Failsafe

On signal loss, the controller enforces a deterministic state machine: attitude leveling followed by linear throttle decay to avoid free-fall [cite_start]and ensure controlled touchdown[cite: 53, 59].

Sim-to-Real Analysis

Policies trained in simulation exhibited smooth descent trajectories but failed to transfer reliably due to latency (exceeding 50 ms), inertial mismatch, and vibration-induced sensor noise. Future work requires domain randomization across mass, noise, [cite_start]and timing[cite: 74, 77].
